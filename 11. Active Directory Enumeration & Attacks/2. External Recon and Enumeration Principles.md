# Active Directory - External Recon & Enumeration Principles

## Why External Recon?
- Validate info from client scoping documents
- Ensure actions are within scope
- Find publicly accessible info (leaked creds, configs, etc.)

---

## What Are We Looking For?

| Data Point | What to Find |
|---|---|
| **IP Space** | ASN, netblocks, cloud presence, DNS records |
| **Domain Info** | Subdomains, mail servers, VPN portals, defenses (SIEM/AV/IDS) |
| **Schema Format** | Email format, AD usernames, password policies → for spraying/stuffing |
| **Data Disclosures** | Public files (pdf, docx, xlsx) with internal links, metadata, credentials |
| **Breach Data** | Leaked usernames/passwords from public breaches |

---

## Where & How to Look

| Resource | Tools/Sites |
|---|---|
| **ASN / IP** | IANA, ARIN, RIPE, BGP Toolkit (bgp.he.net) |
| **DNS / Domain** | Domaintools, viewdns.info, PTRArchive, ICANN, nslookup |
| **Social Media** | LinkedIn, Twitter, Facebook, news articles |
| **Company Websites** | About/Contact pages, embedded docs, intranet links |
| **Cloud/Dev Storage** | GitHub, AWS S3, Azure Blob, Google Dorks |
| **Breach Data** | HaveIBeenPwned, Dehashed |

---

## Key Techniques

### BGP Toolkit
- Find ASN, IP blocks assigned to an org
- Large corps → own ASN | Small orgs → shared infra (Cloudflare, AWS, Azure)
- ⚠️ Confirm scope before testing — shared infra means other orgs could be affected

### DNS Recon
- Use viewdns.info, domaintools for validation
- `nslookup` to verify nameservers
- Find subdomains not in scoping docs but on in-scope IPs → fair game

### Google Dorks
```
filetype:pdf inurl:targetdomain.com       # Find public PDFs
intext:"@targetdomain.com" inurl:targetdomain.com  # Find email addresses
```

### Social Media / Job Postings
- Reveal tech stack, software versions, org structure
- Job listings → e.g., "SharePoint 2013/2016 Admin" = older unpatched versions likely in use

### Username Harvesting
- Tool: **linkedin2username** → scrapes LinkedIn → generates username combos (flast, first.last, f.last)

### Credential Hunting
- Tool: **Dehashed** → search for cleartext passwords/hashes from breach data
- Even old/expired passwords useful for building wordlists
- Try found creds against: Citrix, RDS, OWA, O365, VPN, VMware Horizon

### Finding Leaked Creds in Code
- Tool: **Trufflehog** → scans Git repos for hardcoded secrets
- Site: **Greyhat Warfare** → searches exposed cloud storage buckets

## Enumeration Principles

- Iterative process — repeat multiple times throughout the test
- Start **passive + wide scope** → narrow down → move to **active enumeration**
- Leave no stone unturned; scoping doc + enumeration = primary info sources
- Always save files, screenshots, and tool output immediately

---

## Example Enumeration Flow

1. BGP Toolkit → get IP, mail server, nameservers
2. viewdns.info → validate IP/domain
3. nslookup → verify nameservers, discover new IPs
4. Google Dorks → hunt public files & email addresses
5. Browse company website → contact pages, embedded docs
6. LinkedIn scraping → build username list
7. Dehashed → hunt breach data for valid creds

---

## Key Notes / Reminders
- Always confirm **written scope** before testing any external service
- 3rd party hosted infra (AWS, Azure) may need **prior written approval** from provider
- Even with just **low-privilege domain user creds**, most internal AD enumeration is possible
- Password spraying with harvested external data → common foothold method when stuck
