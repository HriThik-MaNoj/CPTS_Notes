## Bloodhound.py

- Collects AD data from a Linux host using valid domain credentials, then produces **JSON files** you upload to the BloodHound GUI to visualise **attack paths**.

- Data collected includes: users, groups, computers, group memberships, GPOs, ACLs, domain trusts, local admin access, user sessions, RDP access, WinRM access.

#### Two parts

- **BloodHound.py** — the ingestor/collector, runs on Linux
- **BloodHound GUI** — upload JSON files, run queries

**Requirements**

- Valid **domain credentials**
- `impacket`, `ldap3`, `dnspython`

# BloodHound.py – Short Notes

## 1. Execution

```bash
sudo bloodhound-python -u 'forend' -p 'Klmcargo2' -ns 172.16.5.5 -d inlanefreight.local -c all
```

- `-u` → Username
    
- `-p` → Password
    
- `-ns` → Nameserver (Domain Controller)
    
- `-d` → Target domain
    
- `-c all` → Run all collection methods
    

### Output

- Enumerates:
    
    - Domains
        
    - Users
        
    - Computers
        
    - Groups
        
    - Trusts
        
- Generates JSON files:
    
    ```
    <timestamp>_computers.json
    <timestamp>_domains.json
    <timestamp>_groups.json
    <timestamp>_users.json
    ```
