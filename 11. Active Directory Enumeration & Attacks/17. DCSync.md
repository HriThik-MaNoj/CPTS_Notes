- We now have control over user `adunn` who has DCSync privileges in the INLANEFREIGHT.LOCAL domain.

## What the Hell is DCSync and How does it work?
- Technique for stealing the AD password database by using the built-in **Directory Replication Service Remote Protocol** which the DC use to replicate domain data.
- We do this by requesting the DC to replicate passwords via the **DS-Replication-Get-Changes-All** extended right.
- To do this, we need to have access to an account to that has the rights to perform these actions.
- Domain/ Enterprise Admins and default domain admins have this right by default.

#### Using Get-DomainUser to View adunn's Group Membership
```python
PS C:\htb> Get-DomainUser -Identity adunn |select samaccountname,objectsid,memberof,useraccountcontrol |fl
```

- We can use powerview to confirm that this user does have the necessary permissions assigned to their account.
- First, we need to get the user's SID in the above command and then check all ACLs set on the domain object ("DC=inlanefreight, DC=local") using Get-ObjectAcl to get the ACLs associated with the object.

#### Using Get-ObjectAcl to Check adunn's Replication Rights
```python

PS C:\htb> Get-ObjectAcl "DC=inlanefreight,DC=local" -ResolveGUIDs | ? { ($_.ObjectAceType -match 'Replication-Get')} | ?{$_.SecurityIdentifier -match $sid} |select AceQualifier, ObjectDN, ActiveDirectoryRights,SecurityIdentifier,ObjectAceType | fl
```

- DCSync replication can be done using **Mimikatz**, **Invoke-DCSync** and Impacket's **secretdump.py**

# Reversible Encryption 
## What is it?

A UAC flag on user accounts that stores passwords with **RC4 encryption** instead of one-way hashing. Not cleartext — but effectively the same once you have the decryption key.

**The key** is the `Syskey`, stored in the registry. Extractable by a Domain Admin or equivalent.

> [!warning] Any password set while this flag is enabled is recoverable. It won't be re-hashed until the user changes their password.

---

## How it's exploited

Once you dump NTDS (via Domain Admin or DCSync), `secretsdump.py` automatically decrypts any RC4-stored passwords and outputs them in cleartext.

---

## Enumeration

### Find accounts with reversible encryption (Active Directory module)

```powershell
Get-ADUser -Filter 'userAccountControl -band 128' -Properties userAccountControl
```

### Find accounts with reversible encryption (PowerView)

```powershell
Get-DomainUser -Identity * | ? {$_.useraccountcontrol -like '*ENCRYPTED_TEXT_PWD_ALLOWED*'} | select samaccountname,useraccountcontrol
```

### Extract cleartext passwords from NTDS dump

```bash
cat inlanefreight_hashes.ntds.cleartext
# Output: proxyagent:CLEARTEXT:Pr0xy_ILFREIGHT!
```

---

## Key Notes

- The flag is UAC bit `128` (`ENCRYPTED_TEXT_PWD_ALLOWED`)
- Common on service accounts (e.g. proxy agents, legacy app accounts)
- `secretsdump.py` handles decryption automatically during NTDS/DCSync dumps
- To remediate: disable the flag **and** force a password reset — otherwise the old RC4 hash stays
