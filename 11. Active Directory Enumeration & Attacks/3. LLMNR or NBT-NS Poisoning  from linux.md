> Link-Local Multicast Name Resolution (LLMNR) and NetBIOS Name Service (NBT-NS)

### Context

- Phase 1 (unauthenticated enum) complete — users/groups, hosts, DC, naming scheme identified
- Goal: obtain valid cleartext credentials → start credentialed enumeration

---

### Two Techniques (run in parallel)

1. **Network Poisoning** – MitM attack on LLMNR & NBT-NS broadcasts
2. **Password Spraying** – trying common passwords across domain accounts

---

### LLMNR / NBT-NS Poisoning

- Attacker listens for broadcast name resolution requests on the local network
- Poisoned reply tricks victim into sending **NTLM password hash** to attacker
- Can capture **low-priv or admin-level** hashes depending on who triggers it
- Hashes can be:
    - Cracked **offline** → cleartext credentials
    - Used in an **SMB Relay attack** (authenticate to hosts without cracking)

---

### SMB Relay (side note)

- Captured hashes relayed directly to authenticate against domain hosts
- Can grant **admin access** if the account has local admin rights on targets
- Not covered in this module

---

### Outcome

- Valid cleartext domain creds → **foothold established** → proceed to Phase 3 (credentialed enum)

## LLMNR & NBT-NS Primer

- LLMNR and NBT-NS are alternate methods of host identification, used when dns fails.
- LLMNR port : `5355`
- If LLMNR fails NBT-NS is used.
- NBT-NS port : `137`

## LLMNR / NBT-NS Poisoning with Responder

### How the Attack Works

- When LLMNR/NBT-NS is used for name resolution, **any host on the network can reply**
- Attacker uses `Responder` to spoof an authoritative name resolution source in the broadcast domain
- Rogue system pretends to know the location of the requested host → victim communicates with attacker

---

### What Gets Captured

- If the requested host requires **name resolution or authentication**, the victim sends a **NetNTLM hash**
- Hash is subjected to **offline brute force** → attempt to recover cleartext password

---

### What Can Be Done with the Captured Hash

- **Crack offline** → cleartext credentials
- **Relay to another host** – use the auth request to access a different machine
- **Relay to a different protocol** on the same host (e.g. LDAP)

---

### Key Condition for Full Impact

- LLMNR/NBT-NS spoofing + **lack of SMB signing** → can lead to **administrative access** on domain hosts

## Quick Example - LLMNR/NBT-NS Poisoning

Let's walk through a quick example of the attack flow at a very high level:

1. A host attempts to connect to the print server at \\print01.inlanefreight.local, but accidentally types in \\printer01.inlanefreight.local.
2. The DNS server responds, stating that this host is unknown.
3. The host then broadcasts out to the entire local network asking if anyone knows the location of \\printer01.inlanefreight.local.
4. The attacker (us with `Responder` running) responds to the host stating that it is the \\printer01.inlanefreight.local that the host is looking for.
5. The host believes this reply and sends an authentication request to the attacker with a username and NTLMv2 password hash.
6. This hash can then be cracked offline or used in an SMB Relay attack if the right conditions exist.

## Responder – Common Flags & Options

### Modes

- `-A` — **Analyze mode**: passive listening only, shows NBT-NS, BROWSER, LLMNR requests — no poisoning
- Always supply an **interface or IP** when running Responder

---

### Common Flags

|Flag|Purpose|
|---|---|
|`-w`|Start built-in **WPAD rogue proxy** server|
|`-wf`|Start WPAD rogue proxy + **fingerprint** remote host OS/version|
|`-f`|Fingerprint remote host OS and version only|
|`-v`|Increased verbosity — useful for debugging, but noisy|
|`-F`|Force **NTLM authentication**|
|`-P`|Force **proxy authentication**|

---

### Notes on Specific Flags

- `-F` and `-P` — use sparingly, may trigger a **login prompt** on the victim
- `-w` (WPAD proxy) — highly effective in **large organizations**; captures all HTTP requests from IE users with **Auto-detect settings** enabled

**With the above config, responder will listen and answer any requests it sees and writes it to a log file in : `/usr/share/responder/logs` .**

### Good Practice
- Start responder and let it run in a tmux window while we do other enumeration tasks.
- This can maximize the number of hashes obtained.
- Then identify the hashes using [Hashcat example hashes](https://hashcat.net/wiki/doku.php?id=example_hashes) and then use the proper mode to crack them.
- `NTLMv2` hashes are very useful once cracked but `cannot be `used for `pass-the-hash attacks`.

#### Cracking an NTLMv2 Hash With Hashcat
```python
hashcat -m 5600 forend_ntlmv2 /usr/share/wordlists/rockyou.txt
```
