# Active Directory Enumeration — dsquery & LDAP Filtering

---

## 1. dsquery Basics

`dsquery` is a built-in Windows CLI tool that lets you query Active Directory objects directly from PowerShell or CMD.

### Wildcard Search (List Everything in an OU)

```python
dsquery * "CN=Users,DC=INLANEFREIGHT,DC=LOCAL"
```

- The `*` means "match any object type"
- Replace the DN path to target any OU or container
- Returns Distinguished Names (DNs) of every object found

**Example Output:**

```python
"CN=Domain Admins,CN=Users,DC=INLANEFREIGHT,DC=LOCAL"
"CN=krbtgt,CN=Users,DC=INLANEFREIGHT,DC=LOCAL"
```

---

## 2. Combining dsquery with LDAP Filters

You can pass raw LDAP filter strings to `dsquery` using the `-filter` flag — this makes it very powerful.

### Syntax Pattern

```powershell
dsquery * -filter "<LDAP_FILTER>" -attr <attribute1> <attribute2>
```

|Flag|Purpose|
|---|---|
|`-filter`|LDAP filter string|
|`-attr`|Attributes to display in output|
|`-limit`|Max number of results to return|

---

## 3. Practical Filter Examples

### Find Users with PASSWD_NOTREQD Set (UAC flag = 32)

> These accounts don't require a password — a security risk worth noting.

```python
dsquery * -filter "(&(objectCategory=person)(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=32))" -attr distinguishedName userAccountControl
```

### Find All Domain Controllers (UAC flag = 8192)

```python
dsquery * -filter "(userAccountControl:1.2.840.113556.1.4.803:=8192)" -limit 5 -attr sAMAccountName
```

---

## 4. LDAP Filter Syntax — Explained

An LDAP filter string like `userAccountControl:1.2.840.113556.1.4.803:=8192` has three parts:

```python
<attribute> : <OID_matching_rule> : = <value>
```

|Part|Meaning|
|---|---|
|`userAccountControl`|The AD attribute to check|
|`1.2.840.113556.1.4.803`|The OID rule (how to match the bits)|
|`=8192`|The decimal bitmask value to match|

---

## 5. OID Matching Rules

These rules control _how_ bit values are matched against attributes.

|OID|Name|Behaviour|
|---|---|---|
|`1.2.840.113556.1.4.803`|**LDAP_MATCHING_RULE_BIT_AND**|ALL specified bits must be set (exact match for that flag)|
|`1.2.840.113556.1.4.804`|**LDAP_MATCHING_RULE_BIT_OR**|ANY of the specified bits can be set (useful when multiple flags may be present)|
|`1.2.840.113556.1.4.1941`|**LDAP_MATCHING_RULE_IN_CHAIN**|Searches recursively through Distinguished Names — great for nested group membership|

> **Rule of thumb:** Use `.803` when hunting for one specific UAC flag. Use `.804` when any one of several flags is acceptable. Use `.1941` for membership/ownership chain traversal.

---

## 6. Key UAC Bitmask Values

|Decimal|Flag|What It Means|
|---|---|---|
|2|ACCOUNTDISABLE|Account is disabled|
|32|PASSWD_NOTREQD|No password required|
|64|PASSWD_CANT_CHANGE|User can't change their password|
|512|NORMAL_ACCOUNT|Standard user account|
|8192|SERVER_TRUST_ACCOUNT|Object is a Domain Controller|
|65536|DONT_EXPIRE_PASSWORD|Password never expires|

> Values can **stack** — e.g. a value of `66080` = multiple flags OR'd together.

---

## 7. Logical Operators in LDAP Filters

|Operator|Symbol|Usage|
|---|---|---|
|AND|`&`|All conditions must be true|
|OR|`\|`|At least one condition must be true|
|NOT|`!`|Condition must NOT be true|

### AND Example — User with "Password Can't Change" set

```ldap
(&(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=64))
```

### NOT Example — User WITHOUT "Password Can't Change"

```ldap
(&(objectClass=user)(!userAccountControl:1.2.840.113556.1.4.803:=64))
```

### Chaining Multiple Conditions

```ldap
(&(condition1)(condition2)(condition3))
```

---

## 8. Quick Reference Cheatsheet

```powershell
# All objects in a container
dsquery * "CN=Users,DC=DOMAIN,DC=LOCAL"

# Users with no password required
dsquery * -filter "(&(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=32))" -attr distinguishedName

# Find Domain Controllers
dsquery * -filter "(userAccountControl:1.2.840.113556.1.4.803:=8192)" -attr sAMAccountName

# Users with password never expires
dsquery * -filter "(&(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=65536))" -attr sAMAccountName

# Disabled accounts
dsquery * -filter "(&(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=2))" -attr sAMAccountName
```
