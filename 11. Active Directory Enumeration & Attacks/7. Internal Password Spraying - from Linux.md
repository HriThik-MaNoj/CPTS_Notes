## Local Administrator Password Reuse
- CrackMapExec is a handy tool for attempting this attack. It is worth targeting high-value hosts such as `SQL` or `Microsoft Exchange` servers, as they are more likely to have a highly privileged user logged in or have their credentials persistent in memory.

## Password Reuse & Hash Spraying
---

### Password Pattern Reuse

**Local Admin Accounts**

- If local admin password follows a pattern (e.g., `$desktop%@admin123`), try variations on other hosts (e.g., `$server%@admin123`)
- Non-standard local admin accounts (e.g., `bsmith`) may reuse the same password as a domain user with a similar name

**Domain Accounts**

- If user `ajones` password is found, try the same on their admin account (e.g., `ajones_adm`)
- In **domain trust scenarios**, credentials valid in **Domain A** may also work for same/similar usernames in **Domain B**

---

### NT Hash Spraying (Pass-the-Hash)

- If only an **NTLM/NT hash** is retrieved (from local SAM), it can be sprayed across a subnet
- Targets: hosts with the **same local administrator password**

**Key Flag**

|Flag|Purpose|
|---|---|
|`--local-auth`|Attempts login **once per machine** — prevents account lockout|

> ⚠️ **Always use `--local-auth`** when spraying locally — without it, the tool defaults to domain authentication, which can rapidly trigger **domain-wide account lockouts**

---

### Quick Summary

```python
Password found → try pattern variations across hosts
Local user found → check for matching domain user
Domain user found → check for admin account reuse
NT hash found → spray across subnet with --local-auth
```


#### Local Admin Spraying with CrackMapExec

```python
sudo crackmapexec smb --local-auth 172.16.5.0/23 -u administrator -H 88ad09182de639ccc6579eb0849751cf | grep +
```
