
#### Exchange related group membership
- Microsoft exchange, when installed in AD environment, gets highly privileged by default.Two groups matter most here:

### Exchange Windows Permissions

- Not a "protected group" in AD, so it gets less scrutiny
- Members can **write a DACL (Discretionary Access Control List) to the domain object**
- This means an attacker who lands in this group can grant themselves **DCSync privileges** — the ability to replicate all password hashes from the domain controller, just like a legitimate DC would

**How attackers get in:**

- Exploit a DACL misconfiguration
- Compromise an Account Operators member (who can add users to this group)

### Organization Management

- Think of it as **"Domain Admins for Exchange"**
- Can read **all mailboxes** in the domain
- Has **full control** over the OU called "Microsoft Exchange Security Groups" (which contains Exchange Windows Permissions)

> **Why Exchange servers are goldmines:** When you compromise an Exchange server, users who logged into **Outlook Web Access (OWA)** have their credentials cached in memory. Dumping that memory can yield dozens or even hundreds of cleartext passwords or NTLM hashes.
