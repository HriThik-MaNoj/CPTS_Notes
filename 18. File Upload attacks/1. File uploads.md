### Disabling Front-end Validation

#### Concept

Because client-side validation runs entirely within the user's browser, the execution environment is fully controllable. Scripts can be modified, bypassed, or deleted using browser developer tools, eliminating the need to intercept and modify requests using tools like Burp Suite.

#### Methodology

1. **Identify the Target Input Element**
    
    - Open the Page Inspector: `Ctrl + Shift + C` (Windows/Linux) or `Cmd + Option + C` (Mac).
        
    - Click on the file upload button or trigger zone to locate the corresponding HTML code.
        
    - Look for restrictive attributes or event listeners:
        
        HTML
        
        ```
        <input type="file" name="uploadFile" id="uploadFile" onchange="checkFile(this)" accept=".jpg,.jpeg,.png">
        ```
        
2. **Analyze the Validation Script (Optional)**
    
    - Open the Web Console: `Ctrl + Shift + K` (Windows/Linux) or `Cmd + Option + K` (Mac).
        
    - Type the name of the validation function (e.g., `checkFile`) and press `Enter` to view its implementation details and logic.
        
3. **Disable the Client-Side Restrictions**
    
    - Return to the **Inspector/Elements** tab.
        
    - Double-click the event handler attribute executing the validation logic (e.g., `onchange="checkFile(this)"`) and delete it entirely.
        
    - _(Optional)_ Delete the `accept="..."` attribute to display all file types by default in the file selection dialog.
        
    - _Note:_ These modifications are temporary and live only in the current browser session memory; refreshing the page restores the original restrictions.
        
4. **Upload and Execute**
    
    - Select and upload the desired file type (e.g., a PHP web shell) via the modified form UI.
        
    - Use the Page Inspector again to find the rendering path or source URL of the uploaded asset.
        
    - Access the uploaded file directly to interact with the payload:
        
        Code snippet
        
        ```
        http://SERVER_IP:PORT/profile_images/shell.php?cmd=id
        ```

## MIME-Type
- The second and more common type of file content validation is testing the uploaded file's `MIME-Type`. `Multipurpose Internet Mail Extensions (MIME)` is an internet standard that determines the type of a file through its general format and bytes structure.
- Usually done by inspecting the the [File Signature](https://en.wikipedia.org/wiki/List_of_file_signatures) or [Magic Bytes](https://web.archive.org/web/20240522030920/https://opensource.apple.com/source/file/file-23/file/magic/magic.mime).
