- Occurs when a web app exposes a direct reference to an object, which the end user can directly control to obtain access to other similar objects.

## Identifying IDORs
#### URL Parameters & APIs
> e.g. `?uid=1` or `?filename=file_1.pdf`
- We can try incrementing the values of the object references to retrieve other data.
- We can also try fuzzing.

#### Hashing or encoding
> eg: `?filename=ZmlsZV8xMjMucGRm`

- we can identify the encoding type and then decode it and see as `file_123.pdf` and then encode similar object references to try and obtain access to similar objects.
- Sometimes the object reference might appear to be hashed like : (`download.php?filename=c81e728d9d4c2f636f067f89cc14862c`) but if we look at the underlying source code, we might be able to find the unhashed object reference.
