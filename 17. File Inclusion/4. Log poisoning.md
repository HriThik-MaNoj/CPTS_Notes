#### PHP session poisoning
- Most PHP web apps utilise **PHPSESSID** cookies.
- These details are stored in the **session** files inth back-end and saved in **/var/lib/php/sessions** on **Linux** and **C:\Windows\Temp\** on Windows.
- The name of the file that contains the user's data matches the name of our **PHPSESSID** cookie with the **sess_** prefix.
- For example, if the `PHPSESSID` cookie is set to `el4ukv0kqbvoirg7nkp4dncpk3`, then its location on disk would be `/var/lib/php/sessions/sess_el4ukv0kqbvoirg7nkp4dncpk3`

## Server Log Poisoning
- Both `Apache` and `Nginx` maintain various log files, such as `access.log` and `error.log`.
- The **access.log** file contains various information about all requests made to the server, including each request's **User-Agent** header. As we can control the **User-Agent** header in our requests, we can use it to poison the server logs as we did above.
- Then we need to include the logs through LFI. For that we need to have read-access over the logs.
- **Nginx** logs are readable by low privileged users by default.
- **Apache** logs are only readable with high privileges (but in older versions, low privileged users might also have the access to read files.)
- Apache logs : **/var/log/apache2/** on linux and **C:\xampp\apache\logs\** on Windows
- Nginx logs : **/var/log/nginx** on Linux and **C:\nginx\log\*** on windows.
