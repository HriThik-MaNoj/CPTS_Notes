**Port Forwarding**  : Technique that allows us to redirect a communication request from one port to another.

## SSH Local Port Forwarding
![[Pasted image 20260317075625.png]]
- Attack host : 10.10.15.5
- Target : 10.129.15.50
- when scanning the target, we got the following result:
![[Pasted image 20260317075847.png]]
- We can see that ssh is open.
- In order to access the mysql server, there are two options, `ssh` into the target machine and access it from there or `port forward` it to our local port say 1234 and access it locally.

#### Executing the Local Port Forward
```bash
ssh -L 1234:localhost:3306 ubuntu@10.129.202.64
```
- **-L** : Tells the SSH client to request the SSH serer to forward all the data we send via the port 1234 to `localhost:3306`.
#### Confirming Port Forward with Nmap
```bash
nmap -v -sV -p1234 localhost
```

#### Forwarding Multiple Ports
```bash
ssh -L 1234:localhost:3306 -L 8080:localhost:80 ubuntu@10.129.202.64
```

## Setting up to Pivot
- If you type `ifconfig` on the target (ubuntu) host, we'll find that it has multiple NIC's
![[Pasted image 20260317080653.png]]
- First one is the one we connected from our ubuntu machine
- ens224 is communicating with other hosts within a different network.

#### Scanning for hosts in the network that ens224 has access to (172.16.5.0/23)
- We don't know the IP of the hosts within that network, so we'll have to scan the entire subnet first.
- To do this, we will have to perform `dynamic port forwarding` and `pivot` our network packets via the Ubuntu server.
-  We can do this by starting a `SOCKS listener` on our `local host` (personal attack host or Pwnbox) and then configure SSH to forward that traffic via SSH to the network (172.16.5.0/23) after connecting to the target host.
- This is called `SSH tunneling` over `SOCKS proxy`.`
![[Pasted image 20260317082707.png]]
- The attack host starts the SSH client and requests the SSH server to allow it to send some TCP data over the ssh socket.
- The SSH server responds with an acknowledgment, and the SSH client then starts listening on `localhost:9050`.
- Whatever data you send here will be broadcasted to the entire network (172.16.5.0/23) over SSH.
#### Enabling Dynamic Port Forwarding with SSH
```bash
ssh -D 9050 ubuntu@10.129.202.64
```
- **-D** : Enables dynamic port forwarding.
- Now we can use `Proxy chains`, which is capable of routing any tools packets over the port 9050.
- Proxy chains is capable of redirecting TCP connections through TOR, SOCKS and HTTP/HTTPS proxy servers.
- All the traffic will be going thorough the pivot hosts and our attack hosts ip is not revealed.
**To inform proxychains that we must use port 9050, we must modify the proxychains configuration file located at `/etc/proxychains.conf`. We can add `socks4 127.0.0.1 9050` to the last line if it is not already there.**

#### Using Nmap with Proxychains
```bash
proxychains nmap -v -sn 172.16.5.1-200
```


> We can only perform a full **TCP connect scan** over proxychains. If we send partial packets like half connect scans, it will return incorrect results.
