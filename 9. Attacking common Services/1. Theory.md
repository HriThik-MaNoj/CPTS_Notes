## SMB
- we can interact with SMB in windows in multiple ways.
- [winkey] + [R] to open dialogue box and type the share location eg: `\\192.168.220.129\Finance\`
- If anyonymous login in enabled, we wont get password prompt, else we will..
![[Pasted image 20260307104446.png|526]]

#### Windows CMD - DIR

```
dir \\192.168.220.129\Finance\
```

#### Windows CMD - Net Use

```
net use n: \\192.168.220.129\Finance
```

```
net use n: \\192.168.220.129\Finance /user:plaintext Password123
```

#### Command to determine how many files exist in that share

```
dir n: /a-d /s /b | find /c ":\"
```

#### Windows CMD - Findstr

```
findstr /s /i cred n:\*.*
```


#### Windows PowerShell
- we can run windows comments in both cmd and powershell. but we cannot run all cmdlets (powershell commands) in cmd.
```
Get-ChildItem \\192.168.220.129\Finance\
```

```
New-PSDrive -Name "N" -Root
```

#### Windows PowerShell - PSCredential Object

```
PS C:\htb> $username = 'plaintext' PS C:\htb> $password = 'Password123' PS C:\htb> $secpassword = ConvertTo-SecureString $password -AsPlainText -Force PS C:\htb> $cred = New-Object System.Management.Automation.PSCredential $username, $secpassword PS C:\htb> New-PSDrive -Name "N" -Root "\\192.168.220.129\Finance" -PSProvider "FileSystem" -Credential $cred Name Used (GB) Free (GB) Provider Root CurrentLocation ---- --------- --------- -------- ---- --------------- N FileSystem \\192.168.220.129\Finance
```

#### Windows PowerShell - GCI
- We can use the property `-Include` to find specific items from the directory specified by the Path parameter.

```
Get-ChildItem -Recurse -Path N:\ -Include *cred* -File
```

#### Windows PowerShell - Select-String (Uses regex)

```
PS C:\htb> Get-ChildItem -Recurse -Path N:\ | Select-String "cred" -List
```


## Linux

#### Linux - Mount

```
sudo mkdir /mnt/Finance
sudo mount -t cifs -o username=plaintext, password=Password123, domain=.//192.168.220.129/Finance /mnt/Finance
```

**As an alternative, we can use a credential file :** 

```
mount -t cifs //192.168.220.129/Finance /mnt/Finance -o credentials=/path/credentialfile
```

The file `credentialfile` has to be structured like this:

#### CredentialFile

```
`username=plaintext password=Password123 domain=.`
```

> Note: We need to install `cifs-utils` to connect to an SMB share folder. To install it we can execute from the command line `sudo apt install cifs-utils`.
